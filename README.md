# GoldBricks 員工請假系統（免登入）

目前系統啟動時會先進入登入頁，
選擇登入使用者後即可進入系統。

## 使用方式

1. 開啟系統後，先選擇登入使用者
2. 同網址下，系統會自動把資料同步到 Firebase Firestore（`sharedState/main`）
3. 若網路或 Firestore 權限異常，系統會自動退回 `localStorage`（只在同一裝置同一瀏覽器共用）
4. 系統不再提供「右上角提權」或「密碼切換 GoldBricks」功能。


> 注意：系統會自動提供 `GoldBricks` 最高權限管理者帳號（可操作所有功能），其餘員工可由管理員自行新增。


## 管理員如何新增員工

1. 進入「員工管理」→「新增員工」
2. 填寫員工基本資料
3. 儲存後員工資料即加入系統

## Firebase 在本專案中的角色

- 專案仍保留 Firebase 專案設定（`firebaseConfig`），方便後續擴充。
- 目前流程不依賴 Firebase 驗證。

## 看不到人員資料時，先檢查這 3 點

1. **確認是同一個網址與同一個瀏覽器 App**（例如都在同一個 Safari 分頁開啟）。
2. **不要使用無痕模式**（無痕模式常會隔離或清除 `localStorage`）。
3. 若雲端同步失效，本系統會改用瀏覽器 `localStorage` 的 `leaveSystemData`，
   此時只會在「同一裝置＋同一瀏覽器」共用資料；若清除網站資料會回到初始資料。
